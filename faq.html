Preguntas frecuentes

Respuestas directas sobre Open AURA.

¿Qué es Open AURA?

▾
Es un gestor de biblioteca multimedia autoalojado que escanea directorios locales, indexa fotos y videos, genera miniaturas y permite organizarlos en álbumes. Todo corre en tu propia infraestructura.

¿Qué formatos de archivo soporta?

▾
Imágenes: HEIC/HEIF, JPEG, PNG, WebP. Videos: MP4, MOV, AVI, MKV. La conversión de HEIC se realiza al vuelo mediante pillow-heif.

¿Necesito conexión a internet para usarlo?

▾
No. Una vez instalado, funciona completamente offline en tu red local. Solo necesitás internet para la instalación inicial y para actualizaciones.

¿En qué sistemas operativos funciona?

▾
Cualquier sistema que soporte Docker (Linux, macOS, Windows) o Python 3.10+. El frontend Flutter compila para web y escritorio Linux de forma nativa.

¿Cómo funciona el escaneo de archivos?

▾
Seleccionás un directorio desde la interfaz, Open AURA recorre recursivamente todas las carpetas, detecta archivos multimedia por su extensión, genera miniaturas y los indexa en una base de datos SQLite local.

¿Mis archivos originales se modifican?

▾
No. Open AURA es de solo lectura. Nunca modifica, mueve ni elimina tus archivos originales. Solo genera miniaturas y metadatos en su propia base de datos.

¿Soporta múltiples usuarios?

▾
Actualmente está diseñado para uso personal o familiar en red local, sin sistema de autenticación. Para entornos multiusuario se recomienda colocarlo detrás de un proxy inverso con autenticación (nginx, traefik).

¿Cómo hago un backup de mi biblioteca?

▾
Hacé backup de tus archivos originales como siempre. La base de datos de Open AURA (SQLite) y las miniaturas generadas pueden respaldarse aparte, pero son regenerables. Lo importante son tus fotos y videos originales.

¿Puedo acceder desde fuera de mi red local?

▾
Open AURA está pensado para red local. Si necesitás acceso remoto, podés usar un túnel VPN (WireGuard, Tailscale) o exponerlo con un proxy inverso y HTTPS. Evaluá los riesgos de seguridad antes de exponerlo a internet.

¿Tiene aplicación móvil?

▾
Actualmente el frontend compila para web y escritorio Linux. La interfaz web es responsive y funciona correctamente en navegadores móviles. No hay app nativa para iOS/Android en este momento.

¿Cómo puedo contribuir al proyecto?

▾
El código está en GitHub. Podés reportar errores, sugerir funcionalidades o enviar pull requests. Toda contribución es bienvenida.

¿Funciona en Raspberry Pi?

▾
Sí. El backend es liviano (FastAPI + SQLite) y está optimizado para hardware de bajos recursos. La generación de miniaturas puede ser más lenta en una Raspberry Pi, pero el funcionamiento general es fluido.
